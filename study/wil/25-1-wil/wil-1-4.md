# WIL: 이번주 공부 (1월 4주차)

### 이번주 WIL 정리 <a href="#wil" id="wil"></a>

### 배웠던 점

**1. 코드래빗 기반 AI 툴 도입 및 현황** \
이번주 부터 코드래빗 프로를 결제 하여, 디테일한 리뷰를 ai 를 통해 피드백을 받고 있습니다.

기존과 가장 큰 차이점은 다음과 같았습니다.

* **시퀸스 다이어그램 제공:** 구현한 코드를 기반으로 한 시퀸스 다이어그램을 코드래빗이 제공 하여, 제가 의도한 방향에서 많이 벗어난 코드를 구현하였는지 검증이 가능했습니다.
* **모니터링 기능:** 코드래빗이 리뷰한 내용을 통계삼아, 얼마나 개선되고, 개선이 필요한 것들, 어떤 코드 습관에서 피드백을 많이 받는지를 제공 받는 모니터링을 통해 코드 품질에 대해 좀더 신경쓸수있는 정량적 데이터를 확보 하게 되었습니다. (관련 docs : [https://docs.coderabbit.ai/guides/dashboard](https://docs.coderabbit.ai/guides/dashboard))

**2. 클로드 코드 사용 구조 개선 (Claude Code)**

* 명확한 책임 구조 개선 : 기존에 단순히 프론트, 백엔드 에이전트를 각각 나누어서 진행을 하다 보니 컨텍스트가 모호한 작업이 발생한것이 있었습니다. 그렇게 개선의 필요성을 확인하던 와중에 앤트로픽 해커톤에 우승한 개발자의 클로드 코드 사용법 의 오픈소스를 발견하였습니다. 그걸 기반으로 저희 서비스에 맞는 적절한 구조로 바꿔서, 현재는 보다 체계적인 컨텍스트 구조로 개선을 하게 되었습니다.

### 아쉬웠던 점

**1. 코드래빗의 제한적인 리뷰 대상**\
이전까지의 보고 방식으로는 전체 업무 대비 실제 달성률을 명확히 파악하기 어려워 혼란을 겪었습니다.

**2. 클로드 코드 토큰의 효율화**\
목표 달성 실패에 대한 구체적인 대비책이 부족했고, 이슈가 발생했을 때 팀원들에게 전파하는 과정이 매끄럽지 못했던 점이 아쉬웠습니다. 이후 부터는 이런점을 고쳐나가면서 업무 수행을 해야겠다 이해했습니다.
