# WIL: Claude Code 적용

이번주 공부 기간 : 26/01/05 \~ 26/01/11



### 배웠던점

#### Claude code 도입

이번엔 기존에 쓰던 ai 구독을 클로드 코드로 바꿨습니다. 워낙 좋다고 평판을 듣던 서비스 이기도 하고 업무량이 가중되다 보니 한계가 있던 주간이었습니다. 또한 기존 쓰던 구독 서비스가 만료일과 딱 겹치기도 해서 이참에 새롭게 신청하여 사용하게 되었습니다.



#### Skills

Claude 에서 가장 크게 만족하고 썼던것중 하나인  skills 는 ai 에게 지시 및 수행하는 과정에서 명확한 지침이 될수 있는 규칙 같은것이라 이해했습니다. 처음 사용했을때는 스킬도 적용을 안했지만, 점차 복잡한 작업에 대해서는 각 스킬당 50 줄이내에 워크플로우를 지정하였습니다. 그러다 보니 코드의 결과가 만족할만한 수준과 기준에 제가 작성하던 코드와 비슷한 결과물이 나왔습니다.

#### sub agent&#x20;

각각 프론트엔드, 백엔드, 코드리뷰 등에 대한것들을 각각 명령하던 이전 작업에서 이런 작업을 수행해줄 서브 에이전트를 각각 명시하였습니다.&#x20;

이렇다 보니 어떤 특정 작업에 대해서 백엔드와 프론트가 복합적으로 필요한 작업에 대해서 하나의 프롬프트에 명령을 담아도 각 서브에이전트에 명시를 함으로서 병렬로 작업을 할 수 가 있다보니, 작업 시간을 절약할 수 가 있었습니다.



### 아쉬웠던점&#x20;

Claude code Max 를 구독하였지만, 아직 최대한의 효율을 뽑아내지 못하고, 아직은 계속해서 삽질을 하면서 알아가는 과정을 겪고 있습니다.&#x20;

### 기대되는 점

레퍼런스와 참고를 할만한 문서들이 많아서 다양한 접근 및 활용할 수 있는 법에 대해서 적용하는것이 기대가 됩니다.

현재 전사에서 활용할 수 있는 프롬프트 템플릿을 작성하고 있어서, 이것들을 개선하여, 각 컨텍스트에 적용을 할 수 있는 \
템플릿 까지 작성하는 것을 목표로 하고 있기에 다음주차는 연구할 내용이 기대가 되고 있습니다.
